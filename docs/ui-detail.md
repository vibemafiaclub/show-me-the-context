# 5. UI-DETAIL (화면 명세)

**디자인 원칙:** 이모지 사용을 배제하고 ASCII 아트의 구조적 형태와 터미널 색상(Color)을 통해 정보를 밀도 있게 전달.

### 대기 및 세션 선택 화면 (Init Screen) 레이아웃

```text
================================================================================
[ Waiting for Claude Code... ]
================================================================================

   Claude Code 세션이 시작되기를 기다리고 있습니다.
   다른 터미널 창에서 `claude` 명령어를 실행해 주세요.

   (Listening for hooks on http://localhost:3456/hook)

--------------------------------------------------------------------------------
> 이전 세션 불러오기
  종료하기
================================================================================
[Up/Down] 이동   [Enter] 선택
```

**(이전 세션 불러오기 선택 시 전환되는 화면)**
```text
================================================================================
[ 이전 세션 불러오기 ]
================================================================================

> [2026-03-12 15:55] 현재 표시되고있는 context 소모량 bar를 제거하세요.
  [2026-03-11 10:20] Supabase auth 연동을 위한 로그인 페이지 구현
  [2026-03-10 18:05] Vibe Ledger 테이블 스키마 설계 및 Prisma 마이그레이션
  [2026-03-09 09:15] DEUS 프로젝트 구조 초기화 및 패키지 설치

================================================================================
[Up/Down] 이동   [Enter] 해당 세션 시각화 보기   [ESC/q] 대기 화면으로
```

### 메인 화면 (ContextVisScreen) 레이아웃

```text
================================================================================
[ █░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░ ] Context: 15,895 / 200,000 (7.9%)
================================================================================

   ┌──────────┐    System (10,000 tokens)
   │          │    Project Rules, MCP Tools, System Instructions
   │          │
   │          │
   └──────────┘
   ┌──────────┐    User (50 tokens)
   └──────────┘    현재 표시되고있는 context 소모량 bar를 제거하세요.

   ┌──────────┐    Claude (5,445 tokens)
>  │          │    프로젝트 설정 파일을 읽고 statusLine 기능을 비활성화했습니다.
   └──────────┘    .claude/settings.json +3 -4 | Tools: 3 | Thinking: 1500 tokens

================================================================================
[↑/↓] 이동   [Enter] 상세 블록 데이터 보기(Tool JSON)   [ESC] Follow Mode 재개
```

**렌더링 규칙:**

1. **StatusBar:** `(totalTokens / 200000) * 100` 비율에 따라 `█`와 `░` 문자를 조합하여 프로그레스 바 렌더링.
2. **BlockItem Height:** 터미널 가용 높이 대비 각 블록의 토큰 비율(%)로 높이를 비례 계산. 최소 1줄 보장.
3. **Color Coding (박스 테두리 및 텍스트 일부):**
   - System: Blue (파란색)
   - User: Green (초록색)
   - Claude: Yellow/Orange (주황색)
   - 파일 Diff: 추가(`+`)는 Green, 삭제(`-`)는 Red.
